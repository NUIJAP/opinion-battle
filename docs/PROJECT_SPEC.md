## 📋 1. プロダクト概要

### ゲーム名
**立場BATTLE**（仮名）

### コンセプト
「社会的テーマを題材に、ユーザーと AIが立場を選んで対決するスマホゲーム」

- **対象ユーザー**: 20-40代のスマホユーザー（特に日本人）
- **利用シーン**: 通勤・待ち時間・就寝前の「暇つぶし」
- **体験時間**: 1プレイ 60-180秒
- **プレイ回数**: 1日3-5回（ゲーム中毒設計）

### 核となる体験
「AIとの本気の議論対決」が見られる。ユーザーが立場を選んで投票すると、AIが自動で反論を展開。その反論がユーザーの投票に基づいて進化する。

## 🎮 2. コアゲームメカニクス

### 2.1 勝敗判定システム

#### 基本構造
- **ユーザーが選んだ立場のHP** vs **対面するAIのHP**
- 初期HP: 両方とも 100
- HP比較で勝敗が決定

#### HP変動ロジック

```
ユーザーの投票 → AIのHP変化

「いいね」（右スワイプ）
  → ユーザー立場 +15
  → AI立場 HP -5（弱いダメージ）

「参考になる」（上スワイプ）
  → ユーザー立場 +8
  → AI立場 HP -3（微ダメージ）

「反発」（左スワイプ）
  → ユーザー立場 +20
  → AI立場 HP -20（強ダメージ）
  → 次のAI反論が「強気モード」で生成

クイックリプライ選択
  → ユーザー立場 +12
  → 選択内容によりAIの次戦略が変動
```

#### スコア計算

```
基本スコア = 最終HP × 100

ボーナス1（参加者数）= 同時プレイヤー数 × 10
ボーナス2（ラウンド勝）= 勝ったラウンド数 × 50
コンボ倍率 = 1 + (連続勝利数 × 0.5)

最終スコア = (基本スコア + ボーナス1 + ボーナス2) × コンボ倍率

例）
基本: 68 × 100 = 6,800
ボーナス1: 34人 × 10 = 340
ボーナス2: 6勝 × 50 = 300
倍率: 3連勝 = 1.5倍
= (6,800 + 340 + 300) × 1.5 = 10,710点
```

### 2.2 テーマ構造

#### テーマの要素
- **title**: 「AI規制は必要か？」
- **description**: 日本でのトレンドニュース要約（50字程度）
- **stance_a**: 「賛成派」の初期主張（100-150字）
- **stance_b**: 「反対派」の初期主張（100-150字）
- **news_source**: 参考ニュースのURL + 日付
- **difficulty**: 1-5（新規ユーザーは1-2から始まる）

#### テーマ配信スケジュール
- **08:00** : 朝テーマ1つ配信
- **12:00** : 昼テーマ1つ配信
- **20:00** : 夜テーマ1つ配信

毎日新しいテーマ。ユーザーは1日5テーマまで無料プレイ。

## ⚡ 3. ゲームフロー（1プレイの詳細）

### フェーズ 0: テーマ選択（5秒）
```
ユーザーが起動
  ↓
ホーム画面に「今日のテーマ」が大きく表示
  ↓
4択の立場カード表示（各カードに初期主張の一部表示）
  ↓
ユーザーが1つ選択（「賛成派」or「反対派」など）
```

### フェーズ 1: 初期化（0秒）
```
選択した立場: ユーザーHP = 100
対面する立場: AIHP = 100

ゲージ表示開始（画面上部に2本のバー）
```

### フェーズ 2: AIが初手（3-5秒）
```
AI（対面の立場）が初期主張を生成
「字が1文字ずつ出現」（TikTok的な視覚興奮）

例）AI賛成派:
「AIは高度な判断が必要。素人には制御できない。
 だから規制が必須です。」
```

### フェーズ 3: ユーザー反応（ユーザーの速度）
```
ユーザーに3つの選択肢が表示される：

A. 右スワイプ「いいね」
B. 上スワイプ「参考になる」
C. 左スワイプ「反発」

ユーザーが選択
  ↓
即座に「+15点」みたいなアニメで表示
  ↓
AI相手のHP減少アニメ
```

### フェーズ 4: AI反論生成（5-7秒）
```
ユーザーの反応に基づいて、
対面のAIが「次の主張」を自動生成

例）ユーザーが「反発」を選んだ場合、
AI反対派が強気で反論：

「いや、AIは既に医療や金融で活躍してる。
 むしろ規制で失う機会の方が大きい。」
  ↓
AI賛成派のHP -= 20
```

### フェーズ 5: ラウンド繰り返し（1.5-3分間）
```
ラウンド2-7まで同じ流れを繰り返す

毎ラウンド：
- AI主張生成（1-2秒）
- ユーザー投票（任意タイミング）
- HP更新（1秒）

ラウンド数は固定：最大7ラウンド
（早く終わらせたければ「スキップ」ボタンで結果画面へ）
```

### フェーズ 6: 決着・結果表示（約3分後）
```
最終HP比較：
  ユーザー立場: 68
  AI対面立場: 32
  
  「あなたの立場が勝利！」（派手なアニメ）
  スコア: 10,710点 🎉

ランキング表示：
  「今月のランキング：358位 📈」
  「昨日より+24位」

友人比較：
  「銀杏木さんより上！」
  「同じテーマでの最高スコア：8,200点」
```

### フェーズ 7: その後の流れ
```
「もう1回プレイ」ボタン（最重要）
  ↓
自動で次のテーマ提案
  ↓
またはSNS共有画面へ
  ↓
スコアスクショ + 「俺に勝てる？」で友人に送信
```

## 🤖 5. AI自動実行ロジック

### 5.1 バトル中のAI議論生成

```
リアルタイムプレイ時：

ユーザー選択 → 即座に Claude API に送信
┌─────────────────────────────────┐
│ プロンプト（ラウンド実行）      │
├─────────────────────────────────┤
│ テーマ: ${theme}                │
│ ユーザー立場: ${userStance}     │
│ AI立場: ${aiStance}             │
│ これまでの会話: ${battleHistory}│
│ ユーザーの反応: ${userAction}   │
│                                 │
│ → AI主張を150字で生成           │
└─────────────────────────────────┘
  ↓
  Claude が応答（5秒以内）
  ↓
  ユーザーに表示（字が1文字ずつ）
```

### 5.2 学習ループ（毎日23:00）

```
毎日23:00 (Cowork で実行)

1. 過去24時間のバトル結果をSQL で取得
   (最大100件)

2. Cowork で Claude に分析させる
   プロンプト: 「ユーザーが納得した反論パターンは？」
   
3. 結果: JSON で「有効な反論パターンTop3」を抽出
   例）
   [
     { pattern: "経済損失の具体数字を出す", score: 0.87 },
     { pattern: "未来世代への責任を述べる", score: 0.79 },
     { pattern: "最新研究論文を引用", score: 0.74 }
   ]

4. 翌日のAI戦略プロンプトに組み込む
   「ユーザーが過去24時間で納得した反論:
    - 経済損失の具体数字を出す
    - 未来世代への責任を述べる
    こういう反論スタイルを使ってください」
```

### 5.3 「事前計算」の仕組み

```
ユーザーが立場選択
  ↓
その立場×テーマ×複数のシナリオで、
Claude が事前に「複数の反論パターン」を生成
（バックグラウンドで実行）
  ↓
ユーザーの投票速度に関わらず、
すぐに「最適な反論」を返す仕組み

この方法で「インタラクティブな対話」に見える
```

## 💾 6. データベース設計（Supabase）

### テーブル: themes
```sql
CREATE TABLE themes (
  id UUID PRIMARY KEY,
  title VARCHAR(200),                -- "AI規制は必要か？"
  description TEXT,                  -- ニュース概要
  stance_a VARCHAR(500),            -- 賛成派の初期主張
  stance_b VARCHAR(500),            -- 反対派の初期主張
  news_url VARCHAR(500),            -- 参考ニュースURL
  news_date TIMESTAMP,              -- ニュース日付
  difficulty INT (1-5),             -- 難易度
  created_at TIMESTAMP,
  active BOOLEAN DEFAULT true       -- 配信中か否か
);
```

### テーブル: battles
```sql
CREATE TABLE battles (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  theme_id UUID REFERENCES themes(id),
  user_stance VARCHAR(100),         -- "賛成派" or "反対派"
  final_user_hp INT,                -- ユーザーの最終HP
  final_ai_hp INT,                  -- AIの最終HP
  result VARCHAR(50),               -- "win" or "loss" or "draw"
  score INT,                        -- 最終スコア
  rounds_won INT,                   -- 勝ったラウンド数
  battle_history JSONB,             -- ラウンドごとの詳細
  player_count INT,                 -- その時同時プレイ数
  created_at TIMESTAMP,
  played_duration_seconds INT       -- プレイ時間
);
```

### テーブル: users
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY,
  name VARCHAR(100),
  email VARCHAR(255) UNIQUE,
  total_score INT DEFAULT 0,        -- 累計スコア
  win_count INT DEFAULT 0,          -- 勝利数
  loss_count INT DEFAULT 0,         -- 敗北数
  battle_count INT DEFAULT 0,       -- 総バトル数
  current_rank INT,                 -- 月間ランキング
  streak_days INT DEFAULT 0,        -- 連続プレイ日数
  last_played TIMESTAMP,
  pro_member BOOLEAN DEFAULT false, -- Pro会員か
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

### テーブル: ai_patterns（学習用）
```sql
CREATE TABLE ai_patterns (
  id UUID PRIMARY KEY,
  theme_id UUID REFERENCES themes(id),
  pattern_name VARCHAR(200),        -- "経済損失の具体数字"
  effectiveness_score FLOAT,        -- 0.0-1.0
  user_conviction_count INT,        -- 説得できた回数
  last_updated TIMESTAMP
);
```
